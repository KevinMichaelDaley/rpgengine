# Step 1.19: Impact Event Retrieval API

**Parent Epic:** phys-100 (Phase 1: Complete Pipeline)

## Description

Provide API for gameplay systems to retrieve and consume impact events
generated by the physics tick. Impact events are created during cache_commit
(Stage 13) when collision impulses exceed a threshold.

Events are used for:
- Sound effects (collision sounds scaled by impulse)
- Damage systems (physics-based damage)
- Particle effects (sparks, dust)
- Destruction/fracture triggers
- Gameplay callbacks (e.g., "object landed")

## Files to modify

- `include/ferrum/physics/world.h` (add API)
- `src/physics/world/world.c` (implement)
- `tests/physics/impact_event_tests.c` (new)

## API

```c
// Get events from last tick (read-only access)
const phys_impact_event_t *phys_world_get_impact_events(
    const phys_world_t *world,
    uint32_t *out_count
);

// Clear events (call after processing)
void phys_world_clear_impact_events(phys_world_t *world);

// Filter events by body (useful for per-object handlers)
uint32_t phys_world_get_impact_events_for_body(
    const phys_world_t *world,
    pool_handle_t body,
    phys_impact_event_t *out_events,
    uint32_t max_events
);

// Get strongest impact for a body (common case)
bool phys_world_get_strongest_impact(
    const phys_world_t *world,
    pool_handle_t body,
    phys_impact_event_t *out_event
);

// Configure impact threshold
void phys_world_set_impact_threshold(phys_world_t *world, float threshold);
float phys_world_get_impact_threshold(const phys_world_t *world);
```

## Event Lifecycle

```
Tick N:
  1. Physics stages run
  2. Cache Commit (Stage 13) generates events for impulse > threshold
  3. Events stored in world->impact_events buffer

After Tick N (gameplay code):
  4. Gameplay calls phys_world_get_impact_events()
  5. Gameplay processes events (spawn sounds, apply damage, etc.)
  6. Gameplay calls phys_world_clear_impact_events()

Tick N+1:
  7. New events written to now-empty buffer
```

## Acceptance Criteria

- [ ] Events accessible after tick completes
- [ ] Events persist until explicitly cleared
- [ ] Per-body filtering is O(event_count)
- [ ] Threshold is configurable at runtime
- [ ] Buffer grows if needed (arena or realloc)
- [ ] Clearing is O(1)

## Gameplay Integration Example

```c
void gameplay_process_physics_events(game_t *game) {
    const phys_impact_event_t *events;
    uint32_t count;
    events = phys_world_get_impact_events(&game->phys_world, &count);
    
    for (uint32_t i = 0; i < count; ++i) {
        const phys_impact_event_t *e = &events[i];
        
        // Play collision sound scaled by impulse
        float volume = clamp(e->impulse_magnitude / 100.0f, 0.1f, 1.0f);
        audio_play_at(SOUND_IMPACT, e->point, volume);
        
        // Apply damage if impulse high enough
        if (e->impulse_magnitude > DAMAGE_THRESHOLD) {
            entity_t *ent_a = entity_from_phys_body(game, e->body_a);
            entity_t *ent_b = entity_from_phys_body(game, e->body_b);
            float damage = e->impulse_magnitude * DAMAGE_SCALE;
            entity_apply_damage(ent_a, damage);
            entity_apply_damage(ent_b, damage);
        }
        
        // Spawn particles
        particle_spawn_impact(e->point, e->normal, e->impulse_magnitude);
    }
    
    phys_world_clear_impact_events(&game->phys_world);
}
```
