# NPC Barter State Machine: TRADE_INIT, TRADE_SELL, TRADE_BUY

Replace the single-shot `TRADE` tool with a proper engine-managed barter loop. When an NPC enters a trade, it stays in that state until the deal is resolved, rejected, timed out, or combat interrupts it. The barter state is generated by the engine, not the LLM — the LLM only emits tool calls, never arbitrary natural-language negotiation text.

## Current State
The design doc specifies a single `TRADE` tool with three arguments (`customer`, `offer`, `ask`). This is replaced by three smaller tools with at most one argument each, plus a state machine.

## Requirements

### 1. Barter State Machine
The engine owns a `npc_barter_state_t` per participating entity:
```c
typedef enum {
    BARTER_NONE,      /* not in any trade */
    BARTER_PROPOSED,  /* waiting for counter-party response */
    BARTER_ACTIVE,    /* both parties confirmed, exchanging offers */
    BARTER_RESOLVED,  /* deal accepted or rejected, will auto-exit next tick */
} npc_barter_phase_t;

typedef struct npc_barter_state {
    npc_barter_phase_t phase;
    uint64_t           counter_party_id;   /* entity_id of other participant */
    uint64_t           timeout_deadline_us;/* monotonic clock */
    uint32_t           my_offer_item_id;
    uint32_t           my_ask_item_id;
    uint32_t           their_offer_item_id;
    uint32_t           their_ask_item_id;
} npc_barter_state_t;
```
- `counter_party_id` can be another NPC or a player — logic is identical.
- Only one barter state per entity at any time.
- On spawn / state reset: `phase = BARTER_NONE`.

### 2. Tool Set (max 1 argument each)

#### `TRADE_INIT` — 0 arguments
```json
{"name": "TRADE_INIT", "arguments": {}}
```
**Purpose**: Attempt to start a trade loop with the nearest interactable friendly entity.
**Validation**:
1. `phase == BARTER_NONE` — error `ERR_ALREADY_IN_TRADE` if not.
2. Nearest friendly entity within `INTERACTION_RANGE` (2.0m).
3. That entity is also `BARTER_NONE`.
4. `language_similarity >= 0.3`.
5. Neither party in combat.

**Success**: Set both parties to `BARTER_PROPOSED`. The engine-generated barter state (not the LLM) notifies both NPCs on the next turn that they are in a trade loop. The next turn's system prompt for both NPCs will include:
```
You are currently in a trade with <name>. Available tools: TRADE_SELL, TRADE_BUY, TRADE_ACCEPT, TRADE_REJECT.
```
**Error messages returned to LLM**:
- `"TRADE_INIT failed: already in a trade loop."`
- `"TRADE_INIT failed: no friendly entity within 2m."`
- `"TRADE_INIT failed: target is already trading."`
- `"TRADE_INIT failed: language barrier too high."`
- `"TRADE_INIT failed: in combat."`

#### `TRADE_SELL` — 0 or 1 argument
```json
{"name": "TRADE_SELL", "arguments": {"item": "iron_axe"}}
```
**Purpose**: If IN a trade loop, propose/overwrite what the NPC is offering. If NOT in a trade loop, broadcast a sell offer to all nearby friendly NPCs as a one-shot proposal.

**Case A — In trade loop** (`phase == BARTER_ACTIVE`):
- Set `my_offer_item_id` to the item (0 = no offer).
- If counter-party is an NPC, the engine immediately updates their barter state with `their_offer_item_id`.
- Returns: `"You offer <item> in the trade with <name>."`

**Case B — Not in trade loop** (`phase == BARTER_NONE`):
- Broadcast `TRADE_OFFER_BROADCAST` event to all friendly entities within 10m.
- Contains: `seller_id`, `item_id`, `position`.
- Any receiving NPC that calls `TRADE_INIT` in the next turn will see this seller as their nearest friendly entity (priority boost).
- Returns: `"Broadcast sell offer for <item> to nearby friendly NPCs."`

**Error messages**:
- `"TRADE_SELL failed: item not in inventory."`
- `"TRADE_SELL failed: invalid item name."`

#### `TRADE_BUY` — 0 or 1 argument
```json
{"name": "TRADE_BUY", "arguments": {"item": "dried_meat"}}
```
**Purpose**: Mirror of `TRADE_SELL`. If IN a trade loop, sets `my_ask_item_id`. If NOT in a trade loop, broadcasts a want-to-buy offer to nearby friendly NPCs.

**Case A — In trade loop**:
- Set `my_ask_item_id`.
- Update counter-party state.
- Returns: `"You ask for <item> in the trade with <name>."`

**Case B — Not in trade loop**:
- Broadcast `WANT_TO_BUY_BROADCAST` event within 10m.
- Returns: `"Broadcast want-to-buy offer for <item>."`

**Error messages**:
- `"TRADE_BUY failed: invalid item name."`

#### `TRADE_ACCEPT` — 0 arguments
```json
{"name": "TRADE_ACCEPT", "arguments": {}}
```
**Purpose**: Accept the current offer/ask pair and resolve the trade.
**Validation**: Both `my_offer_item_id` and `their_offer_item_id` must be non-zero (or at least one side is non-zero if the other side is zero, meaning a gift). The counter-party must also have called `TRADE_ACCEPT` this turn OR the counter-party is a player (player acceptance is handled via UI, not LLM tool).

**Success**:
- Transfer items between inventories.
- Set both parties to `BARTER_RESOLVED`.
- Next tick: auto-exit to `BARTER_NONE` and log `"Trade with <name> completed."`.

**Error messages**:
- `"TRADE_ACCEPT failed: no offer/ask pair set."`
- `"TRADE_ACCEPT failed: waiting for counter-party."` (only if both are NPCs and the other hasn't accepted yet).

#### `TRADE_REJECT` — 0 arguments
```json
{"name": "TRADE_REJECT", "arguments": {}}
```
**Purpose**: Reject and exit the trade loop immediately.
**Success**: Set both parties to `BARTER_RESOLVED`. Next tick auto-exit. Returns: `"Trade with <name> rejected."`

### 3. Combat Auto-Exit
Any entity entering combat (taking damage, being targeted by ATTACK tool) immediately:
1. Sets its own `phase = BARTER_NONE`.
2. If it had a `counter_party_id`, sets that entity's `phase = BARTER_NONE` too.
3. The next turn's prompt for both will see `"Trade cancelled due to combat."`

### 4. Timeout
`timeout_deadline_us` is set on `TRADE_INIT` success. Default: **120 seconds** (way longer than a normal turn, allowing multi-turn negotiation). If the deadline passes without `TRADE_ACCEPT`, both parties are forced to `BARTER_NONE` and the LLM sees `"Trade timed out."`.

### 5. Engine-Generated Barter State in Prompt
The system prompt builder reads `npc_barter_state_t` and injects exactly this text (no LLM-generated prose):
```
Trade state with <counter_party_name>:
  You offer: <item_name_or_none>
  You want:  <item_name_or_none>
  They offer: <item_name_or_none>
  They want:  <item_name_or_none>
```
If `phase == BARTER_NONE`, no trade section appears.

## Tool Whitelist Updates
The `tool_action` opcode whitelist must replace `TRADE` with:
- `TRADE_INIT   = 0`
- `TRADE_SELL   = 1`
- `TRADE_BUY    = 2`
- `TRADE_ACCEPT = 3`
- `TRADE_REJECT = 4`
- `DEFEND       = 5`
- `ATTACK       = 6`
- `FLEE         = 7`
- `GOTO         = 8`
- `KNOWLEDGE_QUERY = 9`

`SENSE_QUERY` remains async, not in this whitelist.
