Prompt 1: Project Setup and Windowing
Initialize a new Rust project named "ferrum_engine". Update Cargo.toml to include only these minimal dependencies:

gl (OpenGL bindings)

glutin & winit (Window creation and context)

glam (Linear algebra types)

log & env_logger (Minimal logging)

Create the entry point in main.rs. Implement a basic "EngineLoop" struct that initializes a window using winit and an OpenGL 4.6 context using glutin. Enable gl::DEBUG_OUTPUT to print OpenGL errors to the console. Implement a basic main loop that handles WindowClose events and clears the screen to a specific color (e.g., dark gray) every frame. Ensure the code organizes the engine into modules: core, renderer, ecs, physics, net.

Prompt 2: Memory Arena Implementation
In the core module, implement a memory management system based on Arena Allocators. Create a FrameArena struct that wraps a linear/bump allocator. It should:

Pre-allocate a large Vec<u8> or raw byte buffer.

Provide an alloc<T>(&self, value: T) -> &mut T method that copies data into the buffer and returns a mutable reference.

Provide a reset() method to clear the allocator instantly by resetting the pointer. Create a LevelArena struct that functions similarly but is intended for long-lived level data. Ensure these allocators are thread-safe (internal mutability or designed for single-threaded access per worker).

Prompt 3: Fiber-Based Job System
In the core module, implement a Fiber-based Job System. Since we are avoiding heavy external runtimes, implement a JobManager that:

Spawns a pool of worker threads (one per logical core).

Maintains a global WorkStealingQueue (or simple Mutex<Vec<Job>> for simplicity if lock-free is too complex, but prefer lock-free).

Defines a Job trait/struct.

Uses a crate like corosensei (minimal dependency for context switching) OR std::thread::park/unpark to simulate fibers if strict pure Rust is required. Instruction: Implement a "Counter" based synchronization primitive.

Create a function kick_job(job, &counter) and wait_for_counter(&counter). wait_for_counter should yield the current thread/fiber to execute other jobs while waiting, rather than blocking the OS thread.

Prompt 4: Archetype ECS - Core
In the ecs module, implement a Data-Oriented Entity Component System from scratch.

Define Entity as a simple unique integer ID (generation index).

Implement Component trait (marker trait).

Implement Archetype: A struct containing vectors of component data (SoA layout).

Implement World: A struct that manages Entities and a collection of Archetypes.

Implement add_entity which takes a set of components, finds/creates the matching Archetype, and pushes the data.

Implement a Query iterator that allows iterating over all entities containing specific components (e.g., Pos and Vel).

Prompt 5: Modern OpenGL Renderer - DSA & Shader Management
In the renderer module, implement a backend using OpenGL Direct State Access (DSA).

Create a Shader struct that loads GLSL source, compiles, and links.

Create a Buffer struct wrapping glCreateBuffers, glNamedBufferStorage, and glNamedBufferSubData.

Create a Texture struct wrapping glCreateTextures, glTextureStorage2D, and glTextureSubImage2D.

Implement a Mesh struct that holds a VAO (Vertex Array Object) and VBO/IBO.

Ensure all GL calls use the DSA variants (e.g., glProgramUniform instead of glUniform).

Prompt 6: Camera and Input System
In the core module, implement an Input handling system.

Map winit DeviceEvents (MouseMotion, KeyboardInput) to an internal InputState struct (boolean array for keys, float delta for mouse).

Create a Camera struct with position, rotation (Quat), fov, and aspect_ratio.

Implement update_view_proj() logic to calculate View and Projection matrices using glam.

Create a system that updates the Camera transform based on WASD and Mouse input (spectator camera).

Prompt 7: Asset Streaming System
In the core module, implement an Asset Streaming system.

Create a dedicated AssetThread that reads files from disk.

Use channels (std::sync::mpsc) to send requests (e.g., "Load Texture X") to the thread and receive data back.

For textures, implement PBO (Pixel Buffer Object) streaming:

The IO thread loads bytes into a PBO.

The Render thread triggers the DMA transfer from PBO to GPU Texture using glTextureSubImage2D.

Use glFenceSync to ensure data is ready before rendering.

Prompt 8: Terrain - Geometry Clipmaps
In the renderer module, implement the Geometry Clipmap algorithm.

Create a Terrain struct that holds a generic Heightmap texture.

Generate the nested grid geometry (LOD rings) as Vertex Buffers.

Implement the "Toroidal Update" logic: as the camera moves, calculate the offset for the texture sampling coordinates.

Write Vertex and Fragment shaders:

VS: Sample heightmap texture to displace vertices. Implement toroidal wrapping on UVs.

FS: Basic coloring based on height.

Prompt 9: Terrain - Texture Blending & Material Arrays
Enhance the Terrain shader for material blending.

Create a TextureArray (GL_TEXTURE_2D_ARRAY) containing terrain materials (Grass, Rock, Snow).

Implement a "Splat Map" (a low-res texture where RGBA channels correspond to texture indices or weights).

Update the Fragment Shader to sample the Splat Map and blend between the layers in the Texture Array.

Ensure the blending handles boundaries between clipmap levels smoothly.

Prompt 10: Physics - Rigid Body Dynamics
In the physics module, implement a basic Impulse-based rigid body engine.

Define RigidBody component: mass, velocity, angular_velocity, forces.

Define Collider component: AABB and Sphere.

Implement a PhysicsSystem:

Integration Step: Apply gravity and velocity (Semi-Implicit Euler).

Broad Phase: Simple spatial hash or brute force (if count < 1000) to find pairs.

Narrow Phase: Sphere-Sphere and Sphere-Ground collision detection.

Resolution: Apply impulses to separate bodies and conserve momentum.

Prompt 11: Physics - Force Fields
Implement Force Fields as a sub-system of physics.

Define a ForceField component: radius, strength, falloff_type (Linear, Exponential), vector (for directional fields).

Implement a ForceFieldSystem that queries all Dynamic Bodies within the radius of a Force Field.

Apply the calculated force to the RigidBody's accumulator before the integration step.

Prompt 12: Animation - Skeletal Data
In the renderer module, implement skeletal data structures.

Define Bone, Skeleton, and AnimationClip structs.

AnimationClip should hold Keyframes (time, translation, rotation, scale) for each bone.

Implement a function to sample an animation at a specific time t, interpolating between keyframes (Lerp for positions, Slerp for rotations).

Prompt 13: Animation - GPU Skinning
Implement GPU Skinning.

Create an AnimationSystem that computes the Global Pose matrices for a skeleton based on the active animation.

Upload these matrices to a Uniform Buffer Object (UBO) or Shader Storage Buffer Object (SSBO) labeled MatrixPalette.

Modify the standard mesh Vertex Shader to accept ivec4 BoneIndices and vec4 BoneWeights.

Apply the skinning transform in the vertex shader.

Prompt 14: Lighting - Clustered Forward Shading (Setup)
Prepare the renderer for Clustered Forward Shading.

Define a 3D Grid struct (e.g., 16x9x24 clusters).

Implement a Compute Shader compute_clusters.glsl that calculates the AABB (Axis Aligned Bounding Box) for each cluster in View Space.

Dispatch this shader once (or on resize) to populate a ClusterBuffer SSBO.

Prompt 15: Lighting - Light Culling & Rendering
Complete the Clustered Shading pipeline.

Define a Light struct (Position, Color, Radius) and create a random buffer of 1024 lights.

Implement a Compute Shader cull_lights.glsl:

Check intersection between Light spheres and Cluster AABBs.

Write visible light indices to a LightIndexList SSBO and LightGrid texture.

Update the main Fragment Shader to read the LightGrid based on gl_FragCoord, retrieve light indices, and loop through them to calculate lighting.

Prompt 16: Lighting - Static Lightmaps
Implement Static Lightmapping for prop_static entities.

Update the Mesh struct to support a second UV channel (uv2).

Load a "Lightmap" texture (can be a dummy grey texture for now).

Update Fragment Shader to sample the Lightmap using uv2 and multiply it with the Ambient term, adding the Dynamic Light contribution on top.

Prompt 17: AI - Proximity Scheduler
In the ai module, implement a Proximity-based Scheduler.

Implement a Spatial Hash Grid (HashMap<(i32, i32), Vec<EntityID>>) to bucket entities by position.

Create an AISchedulerSystem that runs every frame:

Clear and rebuild the hash grid.

For each AI entity, calculate distance to player.

Assign LOD bucket: High (Update every frame), Medium (Every 5 frames), Low (Every 60 frames).

Only execute the logic systems for entities scheduled for the current frame.

Prompt 18: AI - Utility & Crowd Behaviors
Implement the AI Logic.

Define a UtilityAI component with curves for needs (e.g., Aggression, Survival).

Create a system that evaluates these curves to select a CurrentState (e.g., Attack, Flee, Idle).

Implement a boids-like separation/cohesion force for the Move state to simulate crowd flow without heavy pathfinding.

Execute this logic using the Job System (parallel iteration).

Prompt 19: Networking - UDP & Packet Architecture
In the net module, implement the low-level networking.

Bind a standard std::net::UdpSocket.

Define a Packet enum/struct with a header containing SequenceNumber and AckNumber.

Implement a simple "Reliability Layer":

Keep track of sent packets in a buffer.

On receive, update "Remote Ack".

Resend packets marked "Critical" if not acked within RTT.

Discard "Unreliable" packets if they are older than the last received sequence.

Prompt 20: Networking - State Replication & Compression
Implement World State Replication.

Create a Snapshot struct capturing the state of all networked entities (Position, Rotation).

Implement Delta Compression:

Store the last acknowledged snapshot for each client.

When sending a new snapshot, calculate the difference (Delta) between current and last acked.

Quantize floats to integers (e.g., position * 100 as i16) to save bandwidth.

Serialize and send via UDP.

Prompt 21: Networking - Client Prediction
Implement Client-Side Prediction.

On the client, when Input is pressed, immediately apply physics and store the Input + ResultingState in a HistoryBuffer.

When a Server Snapshot arrives, compare the Server State with the HistoryBuffer state for that tick.

If they differ (Prediction Error), snap to Server State and re-simulate all subsequent inputs in the buffer.

Prompt 22: Dynamic Level Geometry & UI
Implement the final integrations.

Dynamic Geometry: Create a system where a user click modifies the Terrain Heightmap data. Broadcast this "Brush Event" (Location, Radius, Op) to the server, which relays it to all clients. Clients update their Texture/Heightmap.

UI: Implement a basic Immediate Mode UI (or use egui as a permitted dependency if writing a UI library from scratch is too far out of scope, otherwise write a simple textured-quad batcher for text/buttons). Implement an Inventory UI that renders items based on the Inventory component.

Dialog: Parse a simple custom text format (Nodes, Links, Text) and display it via the UI when interacting with NPCs.
